The Results endpoint lets you fetch rich metadata for any publicly accessible web page by passing its URL. Rather than running a full search query, you target a single resource and receive structured information that Google has indexed for it — including the page title, a text excerpt, author attribution, publication date, language, and a representative image URL. Use the `fields` parameter to limit the response to only the properties your application needs.

## Endpoint

```
GET https://www.googleapis.com/customsearch/v1/results
```

## Request Parameters

<ParamField query="url" type="string" required>
  The fully qualified URL of the web page whose metadata you want to retrieve. Must include the protocol (e.g., `https://`). The page must be publicly accessible and indexed by Google.
</ParamField>

<ParamField query="key" type="string" required>
  Your Google API key. You can generate one in the [Google Cloud Console](https://console.cloud.google.com/). Treat this value as a secret and avoid exposing it in client-side code or public repositories.
</ParamField>

<ParamField query="fields" type="string">
  A comma-separated list of response fields to include. When omitted, all fields are returned. Use this parameter to reduce response payload size — for example, `fields=title,url,snippet` returns only those three fields.
</ParamField>

## Response Fields

<ResponseField name="title" type="string">
  The title of the web page as indexed by Google, equivalent to the content of the HTML `<title>` tag.
</ResponseField>

<ResponseField name="url" type="string">
  The canonical URL of the page. This may differ from the URL you submitted if Google has identified a preferred canonical version.
</ResponseField>

<ResponseField name="snippet" type="string">
  A short descriptive excerpt taken from the page content, typically one to three sentences that summarise the page or reflect the most prominent text.
</ResponseField>

<ResponseField name="metadata" type="object">
  Structured metadata extracted from the page's markup, meta tags, and structured data annotations.

  <Expandable title="metadata object">
    <ResponseField name="metadata.author" type="string">
      The name of the page's author or publisher, sourced from meta tags or structured data (e.g., `<meta name="author">`). May be absent if not specified by the page.
    </ResponseField>
    <ResponseField name="metadata.datePublished" type="string">
      The ISO 8601 publication date of the page (e.g., `"2024-03-15"`), sourced from structured data or meta tags. May be absent if the page does not declare a publication date.
    </ResponseField>
    <ResponseField name="metadata.language" type="string">
      The BCP 47 language code of the page's primary content, such as `"en"` or `"fr-CA"`. Detected from the `lang` attribute or content analysis.
    </ResponseField>
    <ResponseField name="metadata.imageUrl" type="string">
      The URL of a representative image for the page, typically sourced from an Open Graph `og:image` tag or a prominent in-page image.
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Metadata availability depends on what the target page declares in its markup. Fields such as `author`, `datePublished`, and `imageUrl` are best-effort and may be absent for pages that do not include the corresponding meta tags or structured data.
</Note>

## Example

<CodeGroup>

```bash curl
curl -G "https://www.googleapis.com/customsearch/v1/results" \
  --data-urlencode "url=https://developers.google.com/custom-search/v1/overview" \
  --data-urlencode "key=YOUR_API_KEY" \
  --data-urlencode "fields=title,url,snippet,metadata"
```

```python Python
import requests

params = {
    "url": "https://developers.google.com/custom-search/v1/overview",
    "key": "YOUR_API_KEY",
    "fields": "title,url,snippet,metadata",
}

response = requests.get(
    "https://www.googleapis.com/customsearch/v1/results",
    params=params,
)
response.raise_for_status()
data = response.json()

print(data["title"])
print(data["metadata"]["datePublished"])
```

```javascript JavaScript
const params = new URLSearchParams({
  url: "https://developers.google.com/custom-search/v1/overview",
  key: "YOUR_API_KEY",
  fields: "title,url,snippet,metadata",
});

const response = await fetch(
  `https://www.googleapis.com/customsearch/v1/results?${params}`
);
const data = await response.json();

console.log(data.title);
console.log(data.metadata?.datePublished);
```

<Tip>
  Use the `fields` parameter in production to request only the properties you need. This reduces response size and improves latency, particularly when fetching metadata for many URLs in sequence.
</Tip>

<Warning>
  This endpoint can only return metadata for pages that Google has indexed. Requests for URLs that are blocked by `robots.txt`, behind authentication walls, or not yet crawled will return an empty or partial response.
</Warning>
