title: "GET /search — Query the Google Custom Search Index"

The Search endpoint is the core of the Google Custom Search API. You send a query string along with your API key and Custom Search Engine (CSE) ID, and Google returns a ranked list of matching web results — complete with titles, snippets, links, and structured page data. Use the pagination parameters to page through result sets and the filtering parameters to narrow results by language, date, or safe-search level.

## Endpoint

```
GET https://www.googleapis.com/customsearch/v1
```

## Request Parameters

<ParamField query="q" type="string" required>
  The search query string. Supports all standard Google Search operators, such as `"exact phrase"`, `-exclusion`, and `site:example.com`.
</ParamField>

<ParamField query="key" type="string" required>
  Your Google API key. You can generate one in the [Google Cloud Console](https://console.cloud.google.com/). Keep this value secret and never expose it in client-side code.
</ParamField>

<ParamField query="cx" type="string" required>
  Your Programmable Search Engine ID (also called the Custom Search Engine ID). Create and manage your search engines at [Programmable Search Engine](https://programmablesearchengine.google.com/).
</ParamField>

<ParamField query="num" type="integer" default="10">
  The number of results to return per request. Accepts values between `1` and `10`. The maximum is `10` per call; use the `start` parameter to retrieve additional pages.
</ParamField>

<ParamField query="start" type="integer" default="1">
  The index of the first result to return, used for pagination. For example, set `start=11` to fetch the second page of results when `num=10`. The maximum allowed value is `91`.
</ParamField>

<ParamField query="lr" type="string">
  Restricts results to documents written in a specific language. Pass a language code prefixed with `lang_`, for example `lang_en` for English or `lang_fr` for French. See the full list of [supported language codes](https://developers.google.com/custom-search/v1/reference/rest/v1/cse/list#languageRestrict).
</ParamField>

<ParamField query="safe" type="string">
  Sets the safe-search filtering level. Accepted values:

  - `active` — filters explicit content from results
  - `off` — disables safe-search filtering

  Defaults to `off` when not specified.
</ParamField>

<ParamField query="dateRestrict" type="string">
  Restricts results to pages indexed within a time window relative to today. Use the following format patterns:

  - `d[N]` — last N days (e.g., `d7` for the past week)
  - `w[N]` — last N weeks
  - `m[N]` — last N months
  - `y[N]` — last N years
</ParamField>

## Response Fields

<ResponseField name="kind" type="string">
  Identifies the resource type. Always returns `"customsearch#search"`.
</ResponseField>

<ResponseField name="queries" type="object">
  Contains pagination metadata for the current request and adjacent pages.

  <Expandable title="queries object">
    <ResponseField name="queries.request" type="array">
      A single-element array describing the current page of results, including `totalResults`, `searchTerms`, `count`, and `startIndex`.
    </ResponseField>
    <ResponseField name="queries.nextPage" type="array">
      A single-element array with parameters for the next page. Absent when you are on the last page.
    </ResponseField>
    <ResponseField name="queries.previousPage" type="array">
      A single-element array with parameters for the previous page. Absent on the first page.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="items" type="array">
  An array of Result objects matching the query. May contain fewer than `num` entries, or be absent entirely, if no results are found.

  <Expandable title="Result object">
    <ResponseField name="items[].title" type="string">
      The HTML-unescaped title of the web page.
    </ResponseField>
    <ResponseField name="items[].link" type="string">
      The full URL of the result page.
    </ResponseField>
    <ResponseField name="items[].snippet" type="string">
      A short text excerpt from the page showing how the query terms appear in context.
    </ResponseField>
    <ResponseField name="items[].displayLink" type="string">
      The display domain of the result (e.g., `"www.example.com"`), without the protocol or path.
    </ResponseField>
    <ResponseField name="items[].pagemap" type="object">
      Structured data extracted from the page, such as `metatags`, `OpenGraph` properties, `Article` schema, or `ImageObject` entries. The shape varies by page.
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  The `queries.request[0].totalResults` field is an estimate. For very broad queries, the actual number of accessible results is capped at 100 (10 pages × 10 results).
</Note>

## Example

<CodeGroup>

```bash curl
curl -G "https://www.googleapis.com/customsearch/v1" \
  --data-urlencode "q=google search api" \
  --data-urlencode "key=AIzaSyD-9tSrke72I6zb8Z9bM4jdO1EXAMPLE" \
  --data-urlencode "cx=017576662512468239146:omuauf_lfve" \
  --data-urlencode "num=2" \
  --data-urlencode "safe=active"
```

```python Python
import requests

params = {
    "q": "google search api",
    "key": "AIzaSyD-9tSrke72I6zb8Z9bM4jdO1EXAMPLE",
    "cx": "017576662512468239146:omuauf_lfve",
    "num": 2,
    "safe": "active",
}

response = requests.get(
    "https://www.googleapis.com/customsearch/v1",
    params=params,
)
response.raise_for_status()
data = response.json()

for item in data.get("items", []):
    print(item["title"], item["link"])
```

```javascript JavaScript
const params = new URLSearchParams({
  q: "google search api",
  key: "AIzaSyD-9tSrke72I6zb8Z9bM4jdO1EXAMPLE",
  cx: "017576662512468239146:omuauf_lfve",
  num: "2",
  safe: "active",
});

const response = await fetch(
  `https://www.googleapis.com/customsearch/v1?${params}`
);
const data = await response.json();

for (const item of data.items ?? []) {
  console.log(item.title, item.link);
}
```

<Tip>
  To paginate through all results, increment `start` by `num` on each request (e.g., `start=1`, `start=11`, `start=21`). Stop when the response no longer includes a `queries.nextPage` field, or when `start` exceeds `91`.
</Tip>
