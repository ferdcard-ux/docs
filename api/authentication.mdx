Every request to the Google Search API must carry a valid credential. The API offers two authentication mechanisms: a lightweight **API key** for server-side and public queries, and **OAuth 2.0** for requests that read or write user-specific data. Choose the method that matches your use case, and make sure you never expose credentials in client-side code or version control.

## API key authentication

An API key is a string token that identifies your project to the API. You can pass it in one of two ways on every request.

<Tabs>
  <Tab title="Query parameter">
    Append `key=YOUR_API_KEY` to the request URL. This is the most common approach for server-to-server calls.

    ```
    GET https://www.googleapis.com/customsearch/v1?q=openai&key=AIzaSyD3oaBxExample1234567890abcdefghijk
    ```
  </Tab>
  <Tab title="Authorization header">
    Send the key as a Bearer token in the `Authorization` header. Use this method when you want to keep credentials out of server logs that record full URLs.

    ```
    GET https://www.googleapis.com/customsearch/v1?q=openai
    Authorization: Bearer AIzaSyD3oaBxExample1234567890abcdefghijk
    ```
  </Tab>
</Tabs>

<Warning>
  Never embed your API key in front-end JavaScript or a mobile app binary. Use environment variables or a secrets manager, and restrict your key to specific IP addresses or referrer URLs in the [Google Cloud Console](https://console.cloud.google.com/apis/credentials).
</Warning>

### Getting your API key

To obtain an API key:

1. Open the [Google Cloud Console](https://console.cloud.google.com/apis/credentials) and sign in.
2. Select or create a project, then navigate to **APIs & Services → Credentials**.
3. Click **Create Credentials → API key**. Your new key appears immediately.
4. Click **Edit API key** to restrict it to the Custom Search API and to specific IP addresses or HTTP referrers.
5. Copy the key and store it in your application's secret manager or environment configuration — never in source code.

### Key parameters

<ParamField query="key" type="string" required>
  Your project's API key. Required on every request unless you are using OAuth 2.0. Obtain one from the Google Cloud Console under **APIs & Services → Credentials**.
</ParamField>

### Code examples

<CodeGroup>

```bash curl
curl -G "https://www.googleapis.com/customsearch/v1" \
  --data-urlencode "q=google search api" \
  --data-urlencode "cx=017576662512468239146:omuauf_lfve" \
  -H "Authorization: Bearer AIzaSyD3oaBxExample1234567890abcdefghijk"
```

```python Python
import requests

response = requests.get(
    "https://www.googleapis.com/customsearch/v1",
    params={
        "q": "google search api",
        "cx": "017576662512468239146:omuauf_lfve",
        "key": "AIzaSyD3oaBxExample1234567890abcdefghijk",
    },
)

response.raise_for_status()
data = response.json()

for item in data.get("items", []):
    print(item["title"], item["link"])
```

```javascript JavaScript (fetch)
const params = new URLSearchParams({
  q: "google search api",
  cx: "017576662512468239146:omuauf_lfve",
  key: "AIzaSyD3oaBxExample1234567890abcdefghijk",
});

const response = await fetch(
  `https://www.googleapis.com/customsearch/v1?${params}`
);

if (!response.ok) {
  throw new Error(`API error: ${response.status}`);
}

const data = await response.json();
data.items?.forEach(({ title, link }) => console.log(title, link));
```

</CodeGroup>

## OAuth 2.0

Use OAuth 2.0 when your application needs to act on behalf of a signed-in Google user — for example, to save searches to a user's account or access personalised results. OAuth 2.0 is not required for anonymous web searches.

### Authorization URL

Direct users to Google's authorization endpoint to begin the OAuth flow:

```
https://accounts.google.com/o/oauth2/v2/auth
```

Include the following query parameters:

<ParamField query="client_id" type="string" required>
  Your application's OAuth client ID, created in the Google Cloud Console under **APIs & Services → Credentials**.
</ParamField>

<ParamField query="redirect_uri" type="string" required>
  The URI Google redirects the user to after authorization. Must match one of the URIs registered for your client.
</ParamField>

<ParamField query="response_type" type="string" required>
  Set to `code` to use the authorization code flow (recommended for server-side apps).
</ParamField>

<ParamField query="scope" type="string" required>
  Space-separated list of OAuth scopes your application requires. See the scopes table below.
</ParamField>

<ParamField query="state" type="string">
  An opaque value you generate to maintain state between the request and the callback. Use this to prevent CSRF attacks.
</ParamField>

### Scopes

| Scope | Access granted |
|---|---|
| `https://www.googleapis.com/auth/cse` | Read and execute Custom Search Engine queries on behalf of the user. |
| `https://www.googleapis.com/auth/cse.readonly` | Read-only access to the user's Custom Search Engine configuration. |

<Tip>
  Request the minimum scope your application needs. Users are more likely to approve a narrow permission request, and you reduce the blast radius if a token is ever compromised.
</Tip>

### Exchanging the authorization code

After the user approves your request, Google redirects to your `redirect_uri` with a `code` parameter. Exchange it for an access token:

```bash curl
curl -X POST "https://oauth2.googleapis.com/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "code=AUTH_CODE" \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET" \
  -d "redirect_uri=YOUR_REDIRECT_URI" \
  -d "grant_type=authorization_code"
```

Use the returned `access_token` in place of your API key for subsequent requests.

## Authentication error codes

<ResponseField name="error.code" type="integer">
  The HTTP status code associated with the authentication failure.
</ResponseField>

<ResponseField name="error.message" type="string">
  A human-readable description of what went wrong.
</ResponseField>

<ResponseField name="error.status" type="string">
  A machine-readable error status string such as `UNAUTHENTICATED` or `PERMISSION_DENIED`.
</ResponseField>

<Expandable title="401 — Missing or invalid API key">
  The request did not include a recognisable credential, or the API key has been deleted.

  ```json
  {
    "error": {
      "code": 401,
      "message": "API key not valid. Please pass a valid API key.",
      "status": "UNAUTHENTICATED"
    }
  }
  ```

  **Fix:** Double-check that you are passing the `key` parameter or `Authorization` header, and that the key value matches what is shown in the Cloud Console.
</Expandable>

<Expandable title="403 — Key restricted or quota exceeded">
  The API key exists but is blocked from making this specific request — either because of an IP/referrer restriction you configured, or because your daily quota is exhausted.

  ```json
  {
    "error": {
      "code": 403,
      "message": "Requests from this IP address are blocked.",
      "status": "PERMISSION_DENIED"
    }
  }
  ```

  **Fix:** Review your key restrictions in the Cloud Console, or see the [Rate Limits guide](/api/rate-limits) if you have hit your daily quota.
</Expandable>
