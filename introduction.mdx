Google is a web platform designed to help you discover, organize, and act on information from across the internet. Whether you're typing a question into the search bar, building a search-powered application, or personalizing your information feed, Google gives you the tools to find exactly what you need — quickly and reliably. This documentation covers everything from basic usage to deep API integration, so you can get value from Google no matter your technical background.

## What you can accomplish with Google

Google is far more than a search bar. The platform brings together a broad set of capabilities that let you interact with web content in powerful ways:

- **Web Search** — Query billions of indexed web pages in milliseconds. Use natural language, keywords, or advanced operators to narrow your results to exactly what you're looking for.
- **Personalization** — Sign in to your Google account to unlock a search experience tailored to your history, preferences, and location. Save searches, star results, and pick up where you left off across devices.
- **Programmatic API Access** — Integrate Google Search directly into your own products using the Search API. Retrieve structured results, images, news, and more with a single HTTP request.
- **Third-party Integrations** — Connect Google with the tools you already use — from productivity suites to data pipelines — through official OAuth flows and webhook support.

## Who this documentation is for

This documentation is organized for two primary audiences:

**End-users** — If you use Google Search directly in your browser, the guides section walks you through features like advanced search operators, SafeSearch controls, and managing your search history. No coding required.

**Developers** — If you're building applications on top of Google's capabilities, the API reference and authentication guides explain how to authenticate, form requests, handle responses, and manage rate limits. Code examples are provided in `curl`, Python, and JavaScript throughout.

<Note>
  Some features described in this documentation — such as API access and personalization endpoints — require a Google account and a valid API key. See the [Authentication](/authentication) guide to get set up.
</Note>
